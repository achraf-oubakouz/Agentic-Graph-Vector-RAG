# Agentic Graph RAG

Agentic Graph RAG is a FastAPI and React application for exploring a document corpus with chunking, embeddings, FAISS retrieval, graph search, and LLM-assisted question answering.

The backend loads a text corpus from `data/data_final_cleaned.txt`, generates analysis artifacts under `static/`, exposes REST endpoints, and can connect to Neo4j for graph-backed workflows. The frontend is a Vite React app for running the pipeline and inspecting retrieval, graph, and agentic results.

## Project Structure

```text
.
|-- app/                    # FastAPI application code
|   |-- core/               # Settings, database/store, CORS/security helpers
|   |-- routers/            # API route modules
|   |-- schemas/            # Pydantic response/request models
|   `-- services/           # Chunking, embeddings, retrieval, graph, LLM logic
|-- colab/                  # Colab notebooks and companion scripts
|-- data/                   # Source corpus loaded on startup
|-- frontend/               # Vite React frontend
|-- k8s/                    # Kubernetes deployment manifests
|-- static/                 # Generated charts, FAISS files, caches, packages
|-- main.py                 # FastAPI entry point
|-- requirements.txt        # Python dependencies
`-- Dockerfile              # Backend container image
```

## Prerequisites

- Python 3.11+ recommended
- Node.js 20+ recommended
- npm
- Optional: Neo4j Aura or another Neo4j instance
- Optional: Gemini API key or local model runtime for LLM synthesis

## Backend Setup

Create and activate a virtual environment:

```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
```

Install dependencies:

```powershell
pip install -r requirements.txt
```

Create a `.env` file in the project root. Keep real credentials out of source control.

```env
APP_NAME=Agentic Graph RAG
APP_VERSION=1.0.0
DEBUG=True

STATIC_DIR=static
CORPUS_PATH=data/data_final_cleaned.txt
FAISS_INDEX_PATH=static/vector_store.faiss
FAISS_METADATA_PATH=static/vector_store_metadata.json
COLAB_PACKAGE_PATH=static/colab_graph_rag_package.zip
RL_POLICY_PATH=static/rl_policy.json
AGENTIC_CACHE_PATH=static/agentic_response_cache.json

NEO4J_URI=
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=
NEO4J_DATABASE=neo4j

ENABLE_LLM_SYNTHESIS=True
LLM_PROVIDER=gemini
GEMINI_API_KEY=
GEMINI_MODEL_NAME=gemini-2.5-flash
```

Start the API:

```powershell
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

Open the API docs at:

```text
http://127.0.0.1:8000/docs
```

## Main API Endpoints

- `GET /` - API health and app metadata.
- `POST /pipeline` - Run chunking, embeddings, and retrieval in one request.
- `GET /corpus/status` - Check whether the corpus is loaded.
- `POST /corpus/upload` - Upload or replace the corpus.
- `POST /chunking` - Compare chunking strategies and generate chunking charts.
- `POST /embeddings` - Build embeddings and generate embedding visualizations.
- `POST /retrieval` - Build/save the FAISS vector store and retrieve matching chunks.
- `POST /query` - Ask a query against the current retrieval state.
- `GET /query/history` - Return previous query history.
- `GET /graph/status` - Check graph connection/status.
- `POST /graph/search` - Search graph data.
- `GET /graph/network` - Return graph network data.
- `GET /colab/status` - Check Colab package readiness.
- `POST /colab/package` - Build the Colab package.
- `POST /agentic/ask` - Ask the agentic RAG workflow.
- `POST /agentic/debug` - Run agentic debug output.
- `POST /agentic/feedback` - Submit feedback for the RL policy.
- `GET /agentic/policy` - Inspect the current policy file.

## Cleanup Notes

The following folders are generated dependency/build artifacts and are usually safe to remove when you want to reclaim disk space:

- `venv/` - Python virtual environment. Recreate with `python -m venv venv` and `pip install -r requirements.txt`.
- `frontend/node_modules/` - npm dependencies. Recreate with `npm install` inside `frontend/`.
- `frontend/dist/` - production frontend build. Recreate with `npm run build`.
- `__pycache__/` and nested `__pycache__/` folders - Python bytecode caches.

Be more careful with:

- `data/` - contains the source corpus.
- `static/` - contains generated but potentially useful demo, cache, FAISS, and Colab package files.
- `.env` - contains local configuration and secrets; do not delete unless you have a backup of the values you need.

Run it locally:

```powershell

```
